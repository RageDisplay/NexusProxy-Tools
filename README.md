# NexusProxy - Tools for Amnezia WG

Автоматическая подготовка сервера:
- Обновление системы (apt-get update/upgrade)
- Установка зависимостей (socat, netcat-traditional, iptables, ufw)
- Создание и настройка UDP-прокси скрипта
- Установка systemd сервиса для автозапуска
- Настройка firewall правил

Управление конфигами:
- Загрузка/сохранение конфигов Amnezia WG (.conf)
- Быстрая замена Endpoint адреса на адрес прокси
- Визуальный редактор конфигурации

Диагностика:
- Проверка SSH соединения
- Логирование всех операций в режиме реального времени
- Статус выполнения команд и отчеты об ошибках

## Требования

- Windows 10/11 с установленным Python 3.8+
- SSH доступ к Linux серверу 
- Интернет соединение для установки зависимостей

## Установка

### 1. Установите Python

Скачайте Python с https://www.python.org/ (3.8 или выше)

При установке обязательно отметьте "Add Python to PATH"

### 2. Установите зависимости

Откройте command prompt в папке проекта и запустите:

```cmd
install_requirements.bat
```

### 3. Запустите приложение

```cmd
run_python.bat
```

## Использование

### Шаг 1: Заполнить данные серверов

В вкладке "Настройка серверов" введите:

ПРОКСИ-СЕРВЕР (сервер, который готовим):
- IP адрес или домен
- SSH пользователь (обычно root)
- SSH пароль
- SSH порт (по умолчанию 22)

ЦЕЛЕВОЙ VPN-СЕРВЕР (куда перенаправляем трафик):
- IP адрес или домен вашего текущего VPN сервера
- UDP порт (одинаковый для обоих серверов, обычно 37884 для Amnezia WG)

### Шаг 2: Проверить SSH соединение

Нажмите кнопку "1. Проверить SSH"

- Приложение попытается подключиться к серверу-прокси
- Проверьте логи в вкладке "Статус"
- Если ошибка - проверьте IP, пароль, firewall

### Шаг 3: Подготовить сервер

Нажмите кнопку "2. Подготовить сервер"

Приложение автоматически:
1. Обновит систему пакеты
2. Установит необходимое ПО (socat и др.)
3. Создаст скрипт прокси в /usr/local/bin/amnezia-proxy.sh
4. Установит systemd сервис для автозапуска
5. Настроит firewall (откроет требуемый порт)
6. Проверит статус сервиса

Ждите окончания (процесс может занять несколько минут).

### Шаг 4: Отредактировать конфиг VPN

В вкладке "Редактор конфига":

1. Нажмите "Загрузить конфиг" и выберите ваш .conf файл
2. Нажмите "Заменить Endpoint" или отредактируйте вручную
3. Замените старый адрес на адрес прокси-сервера
4. Нажмите "Сохранить конфиг"

Примеры замены:

БЫЛО:
```
Endpoint = vpn.example.com:37884
```

СТАЛО:
```
Endpoint = proxy.example.com:37884
```

### Шаг 5: Использовать новый конфиг

Загрузите обновленный конфиг в ваш Amnezia WG клиент.

Трафик теперь будет идти через прокси-сервер вместо прямого подключения.


## Что происходит на сервере

Приложение создает два файла на Linux сервере:

### 1. /usr/local/bin/amnezia-proxy.sh

Скрипт UDP прокси с помощью socat:

```bash
#!/bin/bash
socat UDP-LISTEN:37884,reuseaddr,fork UDP:vpn.example.com:37884
```

- Слушает входящие UDP пакеты на порту 37884
- Перенаправляет их на целевой VPN-сервер
- fork позволяет обрабатывать несколько подключений одновременно

### 2. /etc/systemd/system/amnezia-proxy.service

Systemd сервис для управления прокси:

```ini
[Unit]
Description=Amnezia WG UDP Proxy Service
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/amnezia-proxy.sh
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

- Автоматически запускается при загрузке сервера
- Если прокси упадет - сервис перезагрузит его через 5 секунд
- Логи доступны через: systemctl status amnezia-proxy
